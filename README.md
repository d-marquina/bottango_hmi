# Bottango HMI (tablet)

Flutter app for Android that **replaces Bottango** as the animatronic's
interface. It connects to the Raspberry Pi Pico over **USB (OTG)** and sends
the same serial commands Bottango does, read from an animation exported as
**JSON**.

The Pico's MicroPython driver is **not modified**: the app speaks its protocol.

## Status — MVP (Milestone 1)

- [x] Load an exported animation (Bottango JSON).
- [x] USB serial connection (CDC, 115200 8N1).
- [x] Handshake + effector registration (the "host" role).
- [x] **Mode A** playback (dump): sets the clock to 0 and queues all curves;
      the Pico plays autonomously. Valid for ≤ 8 curves per effector.
- [x] Stop without resetting the board (`xC` command, not `STOP`).
- [ ] Live manual control (sliders) — future.
- [ ] Mode B streaming for long animations (> 8 curves/effector) — future.

## How it builds (nothing installed locally)

You don't need Flutter or the Android SDK on your PC. GitHub Actions builds the
APK:

1. Push this repo to GitHub (`main` branch).
2. Go to the **Actions** tab → **Build APK** workflow → wait ~5 min.
3. Download the **`bottango-hmi-apk`** artifact (contains `app-release.apk`).
4. Copy it to the tablet and install it (enable "Install unknown apps").

To trigger a manual build: Actions → Build APK → **Run workflow**.

## How to use it on the tablet

1. Connect the Pico to the tablet with a **USB-OTG** cable/adapter.
2. Open the app → **Load JSON** (the exported `AnimationCommands.json`).
3. **Connect** → accept the Android USB permission.
4. Tap ▶ on an animation. **Stop** halts it.

## Project layout

```
lib/
├── main.dart                       UI (load, connect, list, log)
├── model/animation_project.dart    JSON parsing + duration
└── serial/
    ├── serial_connection.dart      USB transport ('\n' framing)
    └── bottango_host.dart          host protocol (handshake/setup/play/stop)
test/
└── animation_project_test.dart     parser tests (run in CI)
```

> `android/` is NOT versioned: it is regenerated by `flutter create` in CI.

## Notes

- Flutter is pinned to **3.19.6** in CI: the `usb_serial` plugin uses an older
  Gradle setup (jcenter) that is incompatible with the AGP 9 shipped by the
  latest Flutter. 3.19.6 uses AGP 7 and builds it cleanly.
